Math AA

Math is skill + precision. Marks are lost to tiny mistakes more than “not knowing”.

Papers (SL vs HL)

  • SL: Paper 1 + Paper 2 (2 papers)
  • HL: Paper 1 + Paper 2 + Paper 3 (3 papers)
  • Typically: Paper 1 = non-calc, Paper 2 = calc, Paper 3 = HL-only depth

What Math marks reward

  • Correct method (setup + steps) for method marks
  • Precision (domain restrictions, correct algebra)
  • Communication (clear steps, correct notation)
  • Checking (reasonableness, constraints, final form)

How to improve fastest (the real system)

Do this every session

  • 5 min: quick recall (formulas/ideas) or redo 1 old error
  • 20–40 min: questions (not notes)
  • 10 min: mark + rewrite the clean solution
  • 2 min: Error Log entry (what rule fixes it)

Weekly structure

  • 2 topic drills (single skill focus)
  • 1 mixed set (multiple topics)
  • 1 timed mini-paper (15–30 min)
  • 1 error redo session (redo only mistakes)

Paper strategies

Paper 1 (often non-calc)

  • Algebra must be clean: show steps, don’t jump
  • Write restrictions (domain, denominators, logs)
  • If stuck: try substitution / factoring / graph reasoning
  • Check your final answer fits conditions

Paper 2 (calc allowed)

  • Calculator supports, but doesn’t replace reasoning
  • Use GDC to verify, not to guess blindly
  • Write the math steps anyway (marks aren’t for button presses)
  • Store exact values when possible, round at the end

Paper 3 (HL-only)

  • Longer problems, deeper links across topics
  • Success comes from method + calm persistence
  • Write what you know: define variables, state identities, move forward
  • Check special cases/limits (often reveals the path)

Common mark traps (these destroy grades)

Algebra traps

  • Sign errors when distributing / moving terms
  • Forgetting to square-root ±
  • Incorrect factoring / cancellation
  • Rounding early then compounding error

Domain/validity traps

  • Log arguments must be > 0
  • Denominators cannot be 0
  • Trig inverse outputs have restricted ranges
  • Check if solutions actually satisfy original equation

Trig traps

  • Using degrees vs radians accidentally
  • Forgetting general solution ( + 2πk )
  • Not checking if sin/cos values exceed [-1,1]
  • Mixing identities incorrectly (write them first)

Mini templates (to write in your tests)

Solving equation template

  • State restrictions first
  • Rearrange cleanly
  • Solve and include ± if needed
  • Check solutions against restrictions

Optimization / calculus template

  • Define variables + constraint
  • Write objective function
  • Differentiate, set = 0
  • Second derivative / endpoints check
  • Answer in context + units
