Biology

What the papers test + how to answer like the markscheme wants.

What Bio marks actually reward

  • Correct terminology (keywords, not vague wording)
  • Cause → effect explanations (linked steps)
  • Data interpretation (describe trend + explain with biology)
  • Using examples accurately (not random facts)

Fastest improvement

  • Active recall daily (flashcards/blurting)
  • Practice short-answer markscheme style
  • Train data questions (graphs/tables)
  • Learn common command terms

Papers (what each one expects)

Paper 1A (MCQ)

  • Quick recall + recognition
  • Trap: confusing similar terms (osmosis vs diffusion)
  • Strategy: eliminate two options, then confirm with definition

Paper 1B (data-based / skills)

  • Read graphs/tables, identify trends/outliers
  • Trap: describing without numbers or direction
  • Strategy: “As X increases, Y decreases by __” then explain

Paper 2 (structured + extended)

  • Short answers + longer explanations
  • Trap: writing paragraphs with no keywords
  • Strategy: bullet-quality sentences with the required terms

Paper 3 (if your syllabus has it)

  • Practical/design/data skills
  • Trap: not stating IV/DV/controls clearly
  • Strategy: identify variables + method + reliability + ethics

Command terms (how to not lose marks)

Describe

  • Say what the data shows (trend, direction, numbers)
  • No “why” unless asked
  • Use comparative words: higher/lower, increases/decreases

Explain

  • Give the mechanism: step-by-step cause → effect
  • Use correct terms (enzyme active site, diffusion gradient, etc.)
  • Link back to the question directly

Study plan (Bio = recall heavy)

Daily (10–25 min)

  • Flashcards / blurting on one subtopic
  • Fix 5 “always forget” terms
  • 1 data question (quick graph/table)

Weekly (1–2 hours total)

  • One mixed short-answer set (past paper style)
  • Markscheme rewrite for missed questions
  • Error Log updates (common misunderstandings)

Common mark traps

Content traps

  • Using everyday words instead of biological terms
  • Confusing processes (transcription vs translation)
  • Forgetting where things happen (location matters)
  • Mixing up cause and correlation

Data traps

  • No numbers quoted from the graph/table
  • Ignoring error bars / variability
  • Claiming causation without experimental evidence
  • Not stating anomalies/outliers

IA advice (simple = higher marks)

  • Choose a question with a clear IV and measurable DV.
  • Do repeats (>=5 values or repeats per condition if possible).
  • Control variables explicitly and explain why each matters.
  • Evaluation: limitations + realistic improvements + data quality.
